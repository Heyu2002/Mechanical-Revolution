# 同义词扩展功能 - Synonym Expansion

## 问题描述

用户反馈：不应该需要在 agent 的 `bestFor` 列表中一个一个添加关键词（如"写代码"、"生成代码"、"实现代码"等），系统应该自动理解这些都是编码相关的任务。

**原始问题**：
- "帮我写一个代码" → 需要添加"写代码"
- "帮我生成一个算法" → 需要添加"生成算法"
- "帮我实现一个功能" → 需要添加"实现功能"
- "帮我创建一个脚本" → 需要添加"创建脚本"

这样维护成本太高，而且无法覆盖所有可能的表达方式。

## 解决方案

在 `AgentMatcher` 中实现**同义词扩展（Synonym Expansion）**功能：

### 核心思想

将不同的表达方式映射到核心概念，例如：
- "写"、"生成"、"实现"、"开发"、"创建" → **编码**
- "代码"、"程序"、"脚本"、"函数" → **代码**
- "算法实现"、"写算法"、"生成算法" → **算法**

### 实现机制

1. **定义同义词映射表**：
```typescript
private synonymMap: Map<string, string[]> = new Map([
  // 编码相关
  ["编码", ["写代码", "写", "生成代码", "生成", "实现代码", "实现", "开发", "编写", "创建代码", "创建", "做", "搞"]],
  ["算法", ["算法实现", "算法设计", "算法优化", "写算法", "实现算法", "生成算法", "设计算法", "创建算法"]],
  ["代码", ["代码生成", "代码实现", "代码编写", "代码开发", "代码创建", "程序", "脚本", "函数", "方法"]],

  // 中文相关
  ["中文", ["中文内容", "中文文案", "中文处理", "汉语", "国语"]],
  ["翻译", ["翻译内容", "语言转换", "中英翻译", "英中翻译"]],

  // 图像相关
  ["图像", ["图片", "图像分析", "图片分析", "图像处理", "图片处理", "视觉"]],
  ["识别", ["OCR", "文字识别", "图像识别", "图片识别"]],
]);
```

2. **在分词时进行同义词扩展**：
```typescript
private tokenize(text: string): string[] {
  const tokens: string[] = [];

  // 提取中文字符
  const chineseChars = text.match(/[\u4e00-\u9fa5]/g) || [];
  tokens.push(...chineseChars);

  // 提取英文单词
  const englishWords = text.toLowerCase().match(/[a-z0-9_]+/g) || [];
  tokens.push(...englishWords);

  // 同义词扩展
  const expandedTokens = this.expandSynonyms(text);
  tokens.push(...expandedTokens);

  return tokens;
}
```

3. **扩展逻辑**：
```typescript
private expandSynonyms(text: string): string[] {
  const expanded: string[] = [];

  for (const [core, synonyms] of this.synonymMap.entries()) {
    let matched = false;

    // 检查核心词是否在文本中
    if (text.includes(core)) {
      matched = true;
    }

    // 检查同义词是否在文本中
    if (!matched) {
      for (const synonym of synonyms) {
        if (text.includes(synonym)) {
          matched = true;
          break;
        }
      }
    }

    // 如果匹配，添加核心词的字符（多次添加以增强权重）
    if (matched) {
      const coreChars = core.match(/[\u4e00-\u9fa5]/g) || [];
      // 添加 3 次以增强权重
      for (let i = 0; i < 3; i++) {
        expanded.push(...coreChars);
      }
    }
  }

  return expanded;
}
```

### 权重增强

通过**多次添加核心词字符**来增强匹配权重：

- "帮我写一个代码" → 检测到"写" → 添加 3 次 ["编", "码"]
- "帮我生成一个算法" → 检测到"生成"和"算法" → 添加 3 次 ["编", "码"] + 3 次 ["算", "法"]
- "帮我创建一个脚本" → 检测到"创建"和"脚本" → 添加 3 次 ["编", "码"] + 3 次 ["代", "码"]

这样可以显著提升编码相关任务与 claude-opus-coder 的匹配度。

### 核心算法

#### TF-IDF 向量化

1. **Term Frequency (TF)**: 词在文档中出现的频率
2. **Inverse Document Frequency (IDF)**: 词的稀有程度
3. **TF-IDF**: TF × IDF

#### 同义词扩展增强

通过多次添加核心词字符，增加 TF 值：

```
原始: "帮我写一个代码"
分词: ["帮", "我", "写", "一", "个", "代", "码"]

同义词扩展:
- 检测到"写" → 映射到"编码"
- 添加 3 次 ["编", "码"]

最终: ["帮", "我", "写", "一", "个", "代", "码", "编", "码", "编", "码", "编", "码"]
```

这样"编"和"码"的 TF 值显著增加，提升与 claude-opus-coder 的匹配度。

## 同义词映射设计原则

### 1. 核心概念明确

每个核心词应该代表一个清晰的概念：
- "编码" → 所有与写代码相关的动作
- "算法" → 所有与算法相关的任务
- "代码" → 所有代码相关的名词

### 2. 同义词覆盖全面

包含各种可能的表达方式：
- 正式表达："实现代码"、"编写程序"
- 口语表达："写代码"、"搞个脚本"
- 动词变体："写"、"生成"、"创建"、"开发"

### 3. 避免歧义

确保同义词不会导致误匹配：
- ✅ "写代码" → "编码"
- ❌ "写文章" → 不应该映射到"编码"

### 4. 分层映射

支持多层次的映射：
- 一级："编码" ← "写"、"生成"、"实现"
- 二级："算法" ← "算法实现"、"写算法"
- 组合："写算法" → 同时映射到"编码"和"算法"
